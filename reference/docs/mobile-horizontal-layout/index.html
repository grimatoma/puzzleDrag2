Mobile Landscape — “Maximize the Board” Layout

The board is a fixed 6×6 square, so in landscape it is sized by the shortest available dimension — height. Today a full-width HUD band steals ~15% of that height, and the 44%-wide left column leaves the board height-bound with wasted green margins on either side. The proposal removes the top band and folds the season strip + turns/back into the left rail, handing the board the full viewport height. Flip between the two below.

1Drop the top HUD band

The full-width season strip + back button move into the left rail. Every pixel of reclaimed top height feeds straight into the height-bound board: +— edge, +— area.

2Compact vertical rail

A back ▸ turns ▸ season ▸ stockpile ▸ tools stack at ~30% width (vs 44%). It only needs to be narrow enough that the board stays height-bound — extra width is wasted on a square board anyway.

3Stockpile goes 2-up

In the narrower rail the resource chips stack two columns and scroll — same data, smaller footprint, still glanceable next to the board.

4Board centered, full-bleed height

The playfield fills the entire screen height. The leftover horizontal slack (those green margins) stays as a tidy framed field rather than dead space under a band.

Implementation sketch — this is a CSS-only reshape of the existing BoardLayout template-areas grid (src/ui/puzzleBoard.tsx) plus moving <SeasonIndicator> out of Hud and into the rail. No board/Phaser math changes: computeLayout already sizes tiles to the container it’s handed, so a taller board cell automatically yields bigger tiles. The portrait layout is untouched.
The tile-edge readouts above are measured live from each rendered board, so the size delta is real, not illustrative.
