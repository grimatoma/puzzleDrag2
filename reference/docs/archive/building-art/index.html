The style anchor

Every shipping building imports one shared kit (src/ui/buildings/v2kit.tsx): the same palette, terracotta roof pattern and smoke emitters. That kit — plus a read of all ~30 illustrations — defines the look the pixel set has to hit.

Shared conventions

  • Flat front elevation — centered, whole building visible, no perspective floor
  • Foundation → walls → roof — grey fieldstone base, plaster/brick/timber body, steep terracotta roof + pale ridge cap & dark eave
  • Warm glowing windows — golden light, cross-bar muntins, timber sills
  • Arched timber doors — brass ring/handle, stone keystone
  • Brick chimneys + smoke — furnace / oven glow for the hot trades
  • Cozy storybook props — bread, anvils, flour sacks, hanging signs
  • Soft top-light, single ground shadow

Read & next steps

Consistency held. Generating all five with one shared style preamble + identical params (side view, high detail, detailed shading, selective outline) produced a set that reads as one family: same roof red, cream plaster, warm window glow and top-left light.

What changed vs the SVGs. The pixel sprites add a baked ground shadow and a touch more dimensionality (a “diorama toy” read) where the SVGs are flatter and graphic. The SVGs also animate in‑game — flicker, smoke, turning sails — which these stills don’t capture.

Trade-offs to decide. Pixel art locks resolution and loses the crisp infinite-scale + free recolor/animation the SVG components give. Worth it only if the town wants a unified baked-sprite look (cf. the Stardew-village / TownArtConfig direction).

Outliers for later batches. Four buildings break the cottage mold with sea palettes (harbor dock, lighthouse, sawmill, fishmonger) and two are magical (observatory, portal); chapel/clock-tower/forge use slate, housing uses thatch. Those need their own anchor passes.

How these were made. create_map_object per building, 128×128, transparent. PNGs in assets/png/; the “current” references are the live components rendered to static SVG in assets/svg/.

If we proceed. Next would be tightening per-building props (e.g. bakery pretzel sign, inn tavern sign), then a candidate-of-N pick per building, then wiring into the town manifest in place of the SVG illustrations.
