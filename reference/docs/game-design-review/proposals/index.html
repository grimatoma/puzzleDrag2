Proposals companion · 2026-07

Proposals & mock-ups — what to build, shown before → after

This companion turns the findings report into a set of fully-defined, buildable proposals. Each one pairs a picture of the game as it is with a rendered mock-up of what to build, gives you options where the call is yours, and lands on a concrete file:line implementation spec. It's split into one page per surface so the surfaces can be read — and worked — independently.

Why a separate companion. The findings report answers "what's wrong and why." This answers "here is exactly what it becomes, here's the picture, here are the choices, here's the code to touch." Where a design decision is genuinely yours (how far to pull tile saturation, true multi-column vs. a centered play-frame, name-once vs. abbreviated season strip) the proposal lays out 2–3 options with trade-offs and a recommendation rather than picking for you.

§How to read a proposal

Every proposal is one <article class="prop"> with the same anatomy:

  • ID + severity + effort — carried over from the findings report so you can trace it back (e.g. B1 ↔ the Board tab's candy-square finding).
  • Why — a one-line restatement of the problem with the file:line behind it.
  • Now → Proposed — a real screenshot (where one exists) or a game-accurate HTML mock on the left; the proposed design, rendered with the actual src/tokens.css palette, on the right.
  • Options — 2–3 routes with trade-offs when the choice is yours; the recommended one is flagged.
  • Implementation — the files and functions to change, as a checklist.
The mock-ups are HTML, not sketches. They reuse the game's real tokens (parchment ladder, ember/gold/moss accents, season field gradients, Fraunces display font), so a "Proposed" panel is a believable picture of the change on the real chrome — not a wireframe in a different visual language. They're intentionally schematic (emoji stand in for painterly art) so they stay cheap to iterate; treat them as art-direction, not pixel specs.

Legend: NOW current build · PROPOSED what to build · Recommended the default we'd pick.
