Proposals · Core surface 02

The Town — fully-defined proposals

The town is the game's home screen and its first impression, and the painterly Phaser render is genuinely lovely — but the three gameplay entrances (Farm, Harbor, Mine) look like unbuilt land, a double flower system speckles the whole ground into noise, and the overlay CTAs stack like a debug HUD. Each proposal pairs a picture of the game as it is now with a rendered mock of what to build, and a concrete file:line spec.

Town through-line. The town already grows, animates, and reads as a real place. The work is in the last 15%: (T1) dress the Farm/Harbor beds so the primary entrance looks alive instead of abandoned; (T2) kill the compounding flower speckle so the ground reads as painted, not dithered; (T3–T4) turn the placeholder "Plot" ghosts and the stacked found-settlement boxes into in-world signage; and (T5–T6) fix an inverted CTA hierarchy and a Harbor HUD collision. None is a rebuild — the Phaser scene and tokens are all in place.

T1S1Effort L Top priority #10
Dress each board bed as a living mini-scene — the primary entrance shouldn't read as unbuilt land

The Farm and Harbor boards — the whole reason to enter the puzzle — render as a flat brown dirt pit and a flat blue rectangle. drawBoards fills each bed with a single ground tile (DIRT/WATER), strokes a 2px border and drops a label plate; only the Mine bed gets any props (four ore boulders). Farm gets no furrows or crops, Harbor gets no water motion or boat. Next to the lavishly-detailed cottages and windmill, the entrance to the game looks like the one lot nobody finished. Bed fill + border + plate at src/ui/town/TownScene.ts:688–720; mine boulders (the only dressed bed) at :694.

NOW — real renderdesktop, fresh town
PROPOSEDfurrows + seasonal crop + scarecrow + enter-halo
Options — how alive the beds should be
A · Static dressed bedsRecommended
Furrows + a fixed cluster of crop/boat/cart props + the enter-halo, chosen once per season for the current zone. Pro: ships now, reuses the existing prop-placement path (placeTownTile) and seasonal tile art; big first-impression win for little code. Con: the bed doesn't reflect how full your stockpile is.
B · Stateful bed
Crop density / boat count scales with recent yield, and the bed animates a tiny harvest when you exit a run. Pro: the town visibly reflects play. Con: needs a state feed into the scene and per-frame work; do it as a follow-up to A, not instead of it.
Implementation
  • In src/ui/town/TownScene.ts:683 drawBoards branch per b.kind: for farm, paint a furrow variant into the DIRT bed and scatter seasonal crop props via the existing placeTownTile/season-aware art; for fish, add a moored-boat prop and a foam/ripple tween (reuse the rowboat bob at TownScene.ts:642).
  • Add a persistent "enter" halo: a soft biome-accent ring drawn under the board container (farm = --biome-farm, fish = --biome-fish, mine = --biome-mine), matching the DOM board-entry pills so bed and pill read as one affordance.
  • Keep the mine's boulder scatter (TownScene.ts:694); add one cart prop for parity so all three beds are dressed.
  • Gate the ripple/boat tween behind the scene's existing reduced-motion check so it stays a still image when motion is off.
T2S2Effort S
Stop the compounding flower speckle — one clustered system, not two per-pixel ones

The grass is peppered with tiny multicoloured dots that read as dithering noise, not meadow. Two independent random systems compound: the baked ground paints per-pixel flower specks wherever a hash clears 0.993, and the scene then also stamps a full GRASS_FLOWER tile on 4% of grass cells on top. Together they cover the whole map in evenly-scattered confetti. Per-pixel specks at src/ui/town/proceduralGround.ts:291–293; the 4% tile scatter at src/ui/town/TownScene.ts:562.

NOW — real rendermobile, built-out town
PROPOSEDclustered beds, quieter field
Options — which system survives
A · One clustered tile systemRecommended
Delete the per-pixel specks; keep the tile scatter but cluster it and drop the rate to ~0.012. Pro: one place to tune, reads as flower beds, cheapest baked ground. Con: loses the sub-tile fleck detail (which was the noise anyway).
B · Keep both, tuned
Keep per-pixel specks but raise the threshold well past 0.993 and desaturate; keep the tile scatter but clustered. Pro: retains fine baked-in texture. Con: two knobs to balance forever; easy to drift back to noise.
Implementation
  • Remove (or gate off) the per-pixel flower branch at src/ui/town/proceduralGround.ts:291 — the fh > 0.993 speck write; it is the dominant noise source.
  • In src/ui/town/TownScene.ts:554 scatterGroundDetail, lower the flower rate from 0.04 (line 562) to ~0.012 and cluster: when a cell wins the roll, also flower 2–3 seeded neighbours instead of scattering independently.
  • Desaturate the GRASS_FLOWER palette ~20% (in the flower tile art / C.flower colours) so it recedes into the grass.
  • Add a golden-image assertion on the home-town render so speckle density can't silently regress.

T7S3 / polishEffort S each
Also folded in — smaller town coherence fixes

Lower-severity items found in the same files; each is a small, isolated change.

  • Divergent town renderers — mostly resolved, one stale spot. The report's premise (placement mode drops to a flat SVG grid) doesn't reproduce: the game renders everything — including placement — through the Phaser scene (src/ui/town/TownScene.ts:770 rebuildBuildingsAndPlots draws the green "+ Build" plots in-canvas). The only remaining low-fidelity town is the Dev-Panel wiki's static SVG (src/balanceManager/wiki/EntityVisual.tsx:139 ZoneTownMap), which mirrors buildTownPlan. Fix: either bring that SVG to visual parity or replace it with a baked Phaser snapshot, and add a test asserting the game's placement path never mounts a non-Phaser renderer. (See also T8.)
  • Settlement signs float unanchored. Both the Found and Tier-up signs are absolute top-12 overlays (src/ui/Town.tsx:187), so they hover in dead sky rather than standing in the town — the code comment at :66 even says they "should read as an in-world signpost." Anchor the card to the town-centre plaza in world space (or add a post/stake graphic + ground shadow) so it plants in the map.
  • Default camera is zoomed too far in on portrait. Resting framing uses coverZoom() (fill-the-viewport, no letterbox) at src/ui/town/TownScene.ts:975 restCamera, so on a tall phone the town is cropped and you only see a slice on first load. Offer a gentle "fit-to-content" default (contain, with a small margin) so the whole settlement is legible at rest; the player can still pinch in.
  • Locked board entry is under-communicated. The reason a board is locked lives only in a title attribute (src/ui/Town.tsx:450) — invisible on touch. Surface it as a small lock chip on the pill (e.g. "🔒 Needs the Harbor Dock") or a tap-toast, so mobile players learn why.
  • "Build · N/M plots" polarity is backwards. The label reads Build · {freePlots}/{plotCount} (src/ui/Town.tsx:423), where the numerator is free plots — but "2/3" universally reads as "2 of 3 used". Relabel to "Build · 2 free" or "1/3 built" so the number matches intuition. (Reflected in the T5 proposed pill.)
T8NEWfrom this pass
New findings surfaced while grounding these mockups

Items not in the original report (or where the report is now stale), found reading the town code for this proposal pass. Verify against the live build before scheduling.

  • The "flat SVG-grid fallback in placement mode" finding is stale. Current placement is Phaser-native (src/ui/town/TownScene.ts:747/770 silence the board zones and draw the "+ Build" plots in-canvas while isPlacing); there is no separate SVG grid in the game view. The divergence the report saw now only exists in the Dev-Panel wiki (T7). The S3 should be re-scoped from "game placement downgrades" to "the wiki ships a second, lower-fi town look."
  • The furrow system already exists but never reaches the board beds. The baked ground does draw soil furrows — but only for spec.field parcels (src/ui/town/proceduralGround.ts:295–305), not for the DIRT tiles drawBoards stamps. T1 can reuse that exact furrow math for the Farm bed rather than inventing new art — cheaper than it looks.
  • Board label plates are hard-coded Arial, off the type system. The Farm/Mine/Harbor plates use fontFamily: "Arial" (src/ui/town/TownScene.ts:718), as do the "Plot"/"+ Build" labels (:781) — so the one bit of in-world text ignores the Fraunces/display stack every other surface uses. Route them through the display font when T1/T3 redraw those plates.
  • Villager count ignores town scale. spawnVillagers caps at 4–7 by waypoint count (src/ui/town/TownScene.ts:835); a fully built-out city and a two-plot hamlet feel similarly populated. Consider scaling villager density with tier so growth reads as "busier," reinforcing the settlement-growth fantasy.
