Dynamically-Scaling Pixel-Art UI

How to convert the season turn-tracker (and the app's UI borders) from smooth vector Phaser graphics into crisp pixel art that expands to fit any screen width — with a max size — and how PixelLab generates the assets. With two live, interactive prototypes.

Pixel art has one native resolution. The whole problem of "make it pixel art and let it grow with the screen" comes down to never asking a single bitmap to become a different size by smooth scaling. Instead you split the job in two:

1 · Pixel scale — for crispness

An integer multiplier (×2, ×3, ×4) applied with nearest-neighbor filtering. One source pixel becomes an exact N×N block of screen pixels. This is the only kind of scaling that keeps pixel art sharp. Fractional scales (×2.3) split a pixel across screen pixels → blur and shimmer.

native 16px tall × scale 3 = 48px on screen, every edge razor-sharp.

2 · Layout extent — for fit

To make a thing wider you don't stretch one image. You tile seamlessly repeatable pieces (the sky/ground band), place discrete sprites at anchor points (the season heroes, the wagon), and 9-slice framed boxes (the numeral panel) so corners stay fixed and only the middle repeats.

width = how many tiles fit · max size = clamp the tile count & the integer scale.

The one-line rule. Decide crispness with an integer scale + nearest filter; decide size with tiling + 9-slice + sprite placement. A "max size" is just two clamps: a max width and a max integer scale. Nothing is ever smooth-stretched.

Prototype B — generic pixel-art UI borders (9-slice)

A "9-slice" (9-patch) frame is the reusable answer for every framed UI surface — HUD pills, panels, modals, the numeral box. You author one small frame image; it stretches to any rectangle while the corners stay pixel-perfect and only the straight edges repeat.

The gotcha the verifier flagged. CSS border-image-repeat defaults to stretch — which smears a patterned pixel edge. Use round (scales the tile to fit a whole number of repeats — best for pixel art) or repeat (clips). Flip the toggle above to stretch to see a clean edge turn to mush. The matching trap in Phaser: Phaser.GameObjects.NineSlice stretches its edges (it does not tile) — fine for a 1px-repeat plank, wrong for an ornate motif. For patterned borders on the canvas side, tile the edges yourself or keep the motif in the fixed corners.

Turning a generated frame into a 9-slice (4 steps)

  1. Generate a square frame with a hollow center (create_1_direction_object, 128px). Pick the cleanest of the review candidates.
  2. Measure the insets — how far in from each edge the corner art ends (our wood frame: top:21 right:21 bottom:21 left:21 native px). Store as JSON next to the PNG — see assets/frame-wood.json.
  3. Wire it — DOM: border-image: url(frame-wood.png) 22 round. Phaser: this.add.nineslice(x,y,'frame', null, w, h, 22,22,22,22) (set the texture filter to NEAREST).
  4. Verify edges tile — drag it to extreme sizes; if a motif smears, move that detail into the fixed corner or switch round↔repeat.

Recommendation

Stay in Phaser; flip it to pixel-art mode. The strip already mounts a Phaser game and the board canvas is already pixel art (128px tiles) — moving the strip to pixel art aligns the HUD with the board. Set pixelArt:true + roundPixels:true, render at an integer zoom, build the band from TileSprite layers, the heroes/wagon from Image sprites, and the numeral box from Phaser.GameObjects.NineSlice. The React/Vite wrapper and registry bridge stay exactly as they are.
Three facts verified against the installed source — they shape the build.
  1. pixelArt:true is global & absolute. Phaser's Config.js force-sets antialias:false, roundPixels:true when it's on, silently overriding explicit values. You can't keep antialiased vector heroes beside pixel-art on the same canvas — the procedural petal/leaf/snow particles must be replaced with authored pixel sprites, not reconfigured. (So the migration is all-at-once, not half-and-half.)
  2. NineSlice stretches, never tiles (NineSlice.js L39–44, Phaser 3.90). Put detail in the 4 corners; keep edges/center solid. A repeating-motif border smears. Min size = left+right / top+bottom with no internal clamp — below it the corners invert and garble, so clamp the numeral box's min width (the strip shrinks to 220).
  3. Use scale.mode: NONE, not FIT. The town canvas (TownPhaserCanvas.tsx) is a real pixelArt:true precedent but it uses FIT + zoom:1/dpr = a fractional scale — exactly what blurs pixel art. Render the strip at native px and apply an integer camera.setZoom(S).
Keep the test contract. season-indicator.test.ts renders the SSR fallback in puzzleBoard.jsx (the lazy Phaser child never resolves under renderToString) and asserts data-testid="turns-left", role="status", the count, and aria-label pluralization. Any rewrite must preserve both that fallback and the wrapper's hidden mirror span; frame chrome stays aria-hidden.

  2. Strip art: generate the per-season band tiles + 4 heroes + wagon walk via PixelLab (shared seed, style-locked to the 128px tiles).
  3. Flip the scene: in seasonStripPhaser.tsx set pixelArt:true, roundPixels:true, zoom = integer; rebuild seasonStripScene.ts body with TileSprite + Image + NineSlice instead of Graphics. Keep the registry sync + ResizeObserver untouched.
  4. Generic frame system: add a tiny <PixelFrame> React wrapper (CSS border-image + slice metadata + CSS fallback) and adopt it on panels incrementally.
  5. Verify: visual goldens + mobile width sweep (220→760) + retina (dpr 2/3) crispness check.
Honest trade-off. The current vector strip is smooth and elaborate (gradients, soft particles). Pixel art swaps that for a chunky, cohesive look that matches the board — but a faithful pixel re-creation of every flourish is real art work. Recommended: a cleaner, simpler pixel tracker (band + hero + wagon + numeral) rather than a 1:1 port of every petal. The scaling mechanics here work for either fidelity.

  • Digits: a bitmap glyph atlas at native 26px (fully pixel, scales with S) vs keep Phaser Text (anti-aliasing is acceptable on text, and simpler)?
