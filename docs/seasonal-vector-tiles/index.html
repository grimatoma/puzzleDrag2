Seasonal vector tiles — animation preview

Three board tiles drawn entirely with canvas vector paths (no pixel art), each with a full seasonal lifecycle: four per-season redraws, a subtle seamless idle loop per season, and forward season→season transition morphs played once when a run crosses a season boundary (Spring → Summer → Autumn → Winter). Everything below runs the actual tile-drawing code that would render on the board.

This is a preview before integration. The animation engine wiring exists on the branch, but these tiles take vector precedence over their pixel summer-anchor PNGs only for these three keys — the PNGs stay on disk so we can A/B. Use this page to sign off on the motion first.

What you're looking at

  • Oak (tile_tree_oak) — budding → lush canopy → russet + leaf-fall → bare + snow.
  • Pansy (tile_flower_pansy) — bud → open 5-petal face → fading + seed pod → dormant under snow.
  • Apple (tile_fruit_apple) — blossom → green set → ripe red (hero) → wizened + frost.
  • Idle vs. transition. Idle loops are seamless (driven by a continuous clock) and deliberately subtle — the board sprite adds its own sway, so these stay to canopy shimmer, petal sway, a glint, a drifting leaf/snowflake. Transitions morph colour and shape only (no brightness flash) and are one-shot: there is no Winter→Spring morph, so a new run snaps back to Spring.
  • Engine behaviour mirrored here. In-game a self-detecting controller starts the morph the moment the season flips and settles into the new idle when it completes; under reduced motion the tile just bakes the current-season still. This page plays the same phase timeline.
