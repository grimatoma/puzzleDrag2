Farm Tiles — Animated Pixel-Art Concepts

Five farm tiles reimagined as detailed, characterful 48×48 animations.

The game draws these tiles as static procedural icons (Canvas 2D, ~74 px) that today only get a subtle whole-sprite rotation sway in-game — the artwork itself never moves. This page explores giving each tile a detailed, character-appropriate pixel-art animation instead: the chicken pecks, the carrot's fronds catch the wind, a leaf drifts off the oak. Nothing here is wired into the engine — these are concepts only.

01The five tiles

Each tile is shown large (≈4.5×) on a tile-stage backdrop like the one the game frames sprites against, with a true 1× (48px) view beside it so you can judge how it reads at native size. The motion for each is chosen to suit the object — rigid where a real cob, trunk or root would be; soft only where leaves, fronds and husks are. Hover a palette chip to see its hex.

02Notes

A few things worth flagging before any of these is taken further.

  • All farm-biome tiles. The five belong to the Farm biome. The concepts are 48×48, while the live game renders tiles at ~74 px — adopting one means either upscaling it nearest-neighbour (the crisp pixel look) or redrawing it larger.
  • Motion fits the object. The live game already applies a subtle whole-sprite rotation sway to every tile; these concepts animate the artwork itself, with the motion chosen to suit each object — the cob, trunk and root stay rigid, and only the soft parts (husks, canopy, fronds, leaves) move.
  • Concept only. Nothing here is wired into the engine. This is a direction-finding exploration, not a drop-in asset set.

Reading the 1× view. Each card pairs the big sprite with a true 48px copy because that — not the hero — is roughly the size a tile occupies on the board. Motion that reads clearly at 1× (the chicken's peck, the corn's sliding shine) is the kind worth keeping.

How these were made

The motion is generated procedurally by a small Python / Pillow script, docs/assets/farm-concept/_farm_anim.py, which builds each sprite from multi-step colour ramps, adds spherical shading, and animates per-part articulation (rigid cob/trunk/root; only husks, canopy, fronds and leaves move) before re-rasterising all 18 frames as crisp 48×48 pixel art — re-run it to tune the look. It is a companion to the grass concepts' generator.

This is a concept exploration only — nothing here is wired into the game engine.
