The problem we're fixing: ~25 of 31 features are visible from the first screen. The work is to sequence, gate, hide, and connect existing systems along this timeline.

Context

The game (a Phaser+React drag-chain puzzle, currently themed "The Long Return") has 31 feature slices, ~16 fully wired, plus a deep but out-of-date master doc/story. The core puzzle works: drag a chain of 3+ adjacent same-tiles → every Nth tile (a per-resource UPGRADE_THRESHOLD) spawns an upgrade tile → chains credit resourceProgress → inventory resources + coins; board collapses; 10-turn farm rounds, food-supplied mine/harbor "expedition" rounds.

The problem: features are surfaced all at once; the level/Almanac/map progression is not used to hide content; and the premise's connection points are not wired — farm food doesn't hard-gate mining, and "the next town's buildings cost mining resources" / building specialization was explicitly dropped.

Treat the existing story/keeper/charter content, the master-doc structure (parallel "portfolio", 3-act arc, Hollow-Pact), and all place/character names as OUT-OF-DATE PLACEHOLDERS — not constraints. Only the coded mechanics are current. Theming/names here ("Smoke on the Ridge", "Quarry") are throwaway labels.

Design principles (from external research)

  1. Core loop vs meta loop are different jobs — puzzle drives sessions; town build-up drives retention.
  2. Resources must loop back — every resource needs an outflow so stockpiles don't stall.
  3. Progressive disclosure / enabling — reveal complexity gradually; unlock capability as the player shows competence.
  4. Teach by importance, not complexity, only when needed — one major system per segment.
  5. First 5 minutes: minimal, interactive, fast reward — one short-term + one long-term goal.
  6. Keep the meta lean and mechanical — buildings unlock capability; always leave "room to act" on return.

The stages

Stage 0 · First Chains

Cold open (first ~3–5 min)

Gate: new save. Goal/why: learn the drag, feel the farm respond. Board: only tile_grass_hay + tile_grain_wheat; threshold star markers. Hidden: everything else. Loop: chain → coins + upgrade tile. Pure mastery.

Stage 1 · First Harvest Loop

First building + first order

Gate: first round ends / N coins. Why: "what are resources for?" — orders give targets; building unlocks value. Unlocks: Town view (Hearth + 1–2 plots), Granary (first build, re-costed), Orders; tiles: meadow grass (chain), tile_tree_oak→plank (objective). Loop: chain → resources+coins → build → fill orders → next building.

Stage 2 · A Working Farm

Food surplus + first efficiency

Gate: 1–2 buildings. Why: the farm becomes an engine; first worker eases chains; bread/food foreshadows expeditions. Unlocks: Mill, Bakery (bread), Kitchen (supplies); Workers (first Farmer) + basic tools; tiles birds(eggs)/veg(soup); Quests/XP surfaced. Loop: produces a food surplus — the bridge to mining.

Stage 3 · Smoke on the Ridge

Found the Mine; food gates mining

Gate: farm established + food stockpile. Why: new board + resources; the stockpiled food gains a purpose. Unlocks: map, Mine board (stone→block, iron→iron_bar, coal→coke, gem→cut_gem), "Pack provisions", Mining Camp, Miner, mysterious-ore→rune. New wiring: mine turns = f(food). Loop → two boards: farm food → mine metal.

Stage 4 · Iron & Stone

Mine-resource buildings; tool crossover — the premise's payoff

Gate: have block/iron_bar. Why: metal → better tools → more efficient farming and mining. New wiring: mine-town buildings cost mining resources (Forge: block+iron_bar) + specialization; tool-crossover matrix; Market (caravan_post). Almanac perks L5–L8 come online. This is the complete core loop — the "good playable game" target.

Stage 5 · The Keeper's Bargain

Deferred heavy systems begin

Gate: mine established. First Coexist/Drive-Out → one boon tree + one currency. Keeper as a hazard-to-outlast round (new wiring; auto-prompt). Reskinnable; does not gate the core loop.

Stage 6 · The Harbor & the Network

Third board; the portfolio

Gate: ~Lvl3+ & mine established. Found the Harbor → three-node shared-stockpile network; Tidesinger; pearls→runes→Portal; decorations.

Endgame · defer

The Long Return

3 Hearth-Tokens → Old Capital finale (stub), Charter, sandbox. Out of scope until the core loop is proven.

Authoritative numbers, read from src/constants.ts (UPGRADE_THRESHOLDS, ITEMS_DATA, BUILDINGS, RECIPES, MARKET_PRICES, FARM_TILE_POOL) and src/state/init.ts.

How a tile's two numbers work (today they're ONE). UPGRADE_THRESHOLDS[tileKey] value T does both: tiles-to-upgrade = floor(chainLen / T) upgrade tiles, and tiles-to-resource = every T accrued into resourceProgress = 1 inventory resource (fractional carryover). So they're the same number. Proposal: keep unified early — "every T in a chain = 1 upgrade + 1 stored resource." Chains also pay coins = floor(chainLen × tileValue / 2).

The fixed tutorial spine (scripted & deterministic)

  1. Open (hay+wheat, 150c, 3 orders, scythe×2) → drag a 3+ chain. (card: what/how/why + threshold stars.)
  2. Chain to the 6th hay → first upgrade + hay_bundle. (card)
  3. First order (~5 hay_bundle or 3 flour) → coins + Orders opens.
  4. Objective "3 chains of 5+" → trees (oak→plank) + Town (2 plots).
  5. Build Granary (120c + 8 hay_bundle) → +1 turn.
  6. Objective "store 10 plank" → Mill + birds (eggs) + Workers (Farmer).
  7. Build Mill (150c + 15 plank) → Bakery + Kitchen + veg (soup) + Quests/XP.
  8. Build Bakery (250c + 20 plank) → craft bread (flour×3 + eggs×1 → 125). First real coin engine.
  9. Build Kitchen (200c + 15 plank) → craft supplies (flour×5).
  10. Objective "pack 3 supplies" → map + found the Mine (Stage 3).

After step 10 the scripted spine ends; Stage 4+ is player-directed (the registry still gates).

Worked economy (so the numbers feel real)
  • 1 flour = 5 wheat tiles; 1 egg = 6 bird tiles.
  • 1 bread = 3 flour + 1 egg = 15 wheat + 6 bird tiles → sells 125 (Mill: 10 wheat + 6 birds); raw 3 flour = 24c → Bakery ≈ 5× multiplier (the early inflection point).
  • 1 mine trip (3 turns) = 3 supplies = 15 flour = 75 wheat tiles (or 3 bread) — the throttle. Tuning flag: 75 may be steep → supplies=flour×3 (45 tiles), Mining Camp +1, or a bigger starter ration.
  • Granary (120c): chains pay only 3–5c each → orders + the "build 1 building" quest (100c) + 150 starting coins get there in ~2–3 orders. Coins come from selling refined goods, not raw chaining.
Two worker bugs to fix (the "hire for efficiency" loop is broken today):
  • Miner targets the wrong category — threshold_reduce_category: "wood" speeds tree-felling, not mining. Fix: point at the stone/mine category.
  • First hire ~inert — reductions fold as amount × hired / maxCount, so hire #1 shaves ~0.1 tile. Fix: make hire #1 a clear −1 (step the curve / lower maxCount). (Verify features/workers/aggregate.ts.)

Gap features / missing connection points (what to build)

  1. Unified Progression/Unlock registry (src/progression/unlocks.*) — the backbone. Maps every gateable thing (views/modals, buildings, tiles, workers, recipes, features) to a condition (level/flag/buildingBuilt/settlementFounded/resourceCount/priorUnlock/stage). Mirror the tile discovery pattern. One isUnlocked(state,key); UI layers consult it (src/ui.*, Town picker, cartography, tileCollection, workers, crafting). The single change that hides everything-at-once.
  2. Hard farm→mine food gate (EXPEDITION/DEPART + EXPEDITION_FOOD_TURNS; block zero-food).
  3. Mine-resource building costs + specialization (reverse "all buildings everywhere").
  4. Guided objective spine (ordered main objectives that flip unlock flags).
  5. Tool crossover matrix (finish per-biome tool effects).
  6. Onboarding as system-intro cards (what/how/why per first unlock; reuse tutorial slice).
  7. Keeper auto-prompt + Drive-Out-as-round (Stage 5).

Implementation approach (phased)

Build the mechanism first, then pour staged content through it. Use the subagent-driven-development skill per CLAUDE.md.

  • Phase A — Unlock backbone. Add src/progression/unlocks.* + isUnlocked(); bump SAVE_SCHEMA_VERSION; UI consults it (register all as unlocked first, then move gates).
  • Phase B — Stage gates 0–2. Hide all but board+2 tiles at fresh save; reveal on the objective spine; add objectives track + first-time tutorial cards; trim FARM_TILE_POOL; re-cost buildings.
  • Phase C — Farm→mine connection (3–4). Hard food gate; found-the-mine flow; mine board reveal; mine-resource building costs + specialization; tool crossover; Market; fix worker bugs. Delivers the complete core loop.
  • Phase D — Deferred meta (5–6). Gate keepers/boons/currency; harbor + portfolio; portal/decorations.
  • Phase E — Endgame stub left as-is.

Priority A–C. Touch-points: src/state.*, src/constants.ts, src/state/biomeAccess.*, src/features/{cartography,zones,tileCollection,workers,crafting}/*, src/ui.*, src/router.*, src/features/tutorial/*.
