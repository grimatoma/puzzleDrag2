The growing town

Roads first, buildings front the street. A road is never routed to a building; cottages sit beside it at a setback, so streets always run between them.
Bounded plaza + farm parcel. The village green (well→fountain→monument) is a fixed hub; the farm is a separate fenced field beside it — no overlap, no road through it.
Roads grow → new routes. Outpost main street → Hamlet extends it → Village opens a back lane & bridges the river → City knits blocks with cross-lanes + a second bridge, and cobbles the centre.
A river gives the town a reason & a pulse. Settlement nestles in the river's elbow; the dock + fishing spot, ducks, and reeds make the water lived-in, not scenery.
Dressing scales with status. Sparse, slightly raw Outpost → lamps & market at Village → banners, festoon lights, monument & second market at City. Each upgrade visibly earns more life.
People make it a place. A fisher casts at the dock, a vendor tends the stall, kids chase round the fountain, a farmer works the field, a dog trots the high street. Density rises every rung.

Tile & asset inventory

Every tile, transition, prop and overlay this scene needs — the generation backlog (we'll make the pixel art via PixelLab later). Autotile sets (roads, rivers, terrain seams) are shown as one entry per set with a representative sketch; the description lists the pieces. Each entry has size (in 32px tiles), how many variations to draw for natural variety, and whether it's animated.

Tileset bible — decide these once, before drawing (they keep 176 hand-made tiles reading as ONE set):
  • Palette: one shared master ramp; cool shadows / warm highlights. Each material (grass·dirt·cobble·flag·water·sand·farm·wood·stone) gets a 4–5 step ramp sampled from it. Anchor style = the cozy summer palette already used by the board tiles.
  • Light: single global direction top-left. Corners/T-junctions of lit roads must be redrawn per rotation (you can't rotate a top-left light) — budget the full case set, not "1 of 4".
  • Outline & density: dark selective outline (selout) on objects, none between adjacent ground tiles; 32px base, no sub-tile detail you can't actually paint at 32px.
  • Anchor / depth: every sprite has a bottom-centre origin + a y-sort line so characters pass behind/in front correctly. Contact shadow baked under every elevated sprite (~22%, top-left) — see FX category.
  • Collision: each tile/object carries a collision flag/mask (buildings, fences, water, trees, boulders solid) so foot traffic routes around them.
  • Autotile model: surfaces & terrain seams = 16-tile Wang 2-corner sets (edges + all 4 outer + 4 inner corners + diagonals); roads/rivers = 16-piece connectivity (straight·corner·T·cross·end). Each "set" entry below = ~16 real tiles, so the true tile count is ~300–500, not 176.
  • Animation: every animated entry needs frame-count + fps + loop type at art time; characters are 4-directional walk × N frames (the most expensive assets).
  • Season policy: the town renders summer-only for now (the board has its own seasonal engine). If the town should reflect seasons later, the 12 ground terrains + trees + roofs need winter/autumn recolours — flagged, not yet specced.
  • Day/night policy: a day/dusk toggle exists → the FX · night layer (lit-window overlays, lamp light-pools, ambient tint) is in-scope; if night is cut, drop that category.

Why so many transition tiles? Two materials must never butt edge-to-edge — every boundary (grass↔dirt, grass↔water, dirt↔cobble…) needs a Wang/blob set so the world reads as one painted place. Roads & rivers use a thin 16-piece connectivity set (crisp junctions); surfaces use a blob set (soft borders). That seam-handling is most of the believability. See the Art bible tab for the global palette / light / autotile rules.

Art bible

Decide these once, before any pixel is drawn — they are what make 176 separately-authored tiles read as one cohesive set instead of a patchwork. This is the contract every tile/prop/character in the inventory must obey.

1 · Master palette

One shared, limited palette across the whole set: cool, desaturated shadows → warm, saturated highlights (hue-shift, don't just darken). Each material gets a 4–5 step ramp sampled from the master palette. Anchor the hue family to the cozy summer board tiles already in the game.

2 · Light & shading

Single global light: top-left. Every sprite is lit from the upper-left; the lower-right is in shade. Be consistent across all 176 tiles.
Corners can't rotate. A road corner / T-junction lit from top-left does NOT look right rotated 90° — redraw each rotation. Budget the full case set, not "1 of 4".
Hue-shift, no pillow-shading. Shadows shift cooler/bluer, highlights warmer; never just darken the base. No concentric "pillow" gradients — light a form, not an outline.

3 · Outline & resolution

32px base tile. Don't draw detail you can't actually paint at 32px; the diagonal river uses interpolated ½/⅓/⅔ corner tiles, not sub-pixel fades.
Selective outline (selout). Objects (buildings, props, trees, characters) get a dark selective outline; adjacent ground tiles have NO outline between them, so terrain reads as continuous.
Variety beats repeat. The most-seen tile (grass) needs ≥8 variants or a scatter-decal layer; large flat fills are the #1 place tiling repeat shows.

4 · Anchors, depth & shadow

Bottom-centre origin. Every sprite anchors at the bottom-centre of its footprint cell.
Y-sort line. Tall objects (trees, buildings, lamps) carry a y-sort baseline so characters pass correctly in front of / behind them.
Contact shadow, baked. A soft ellipse shadow (~22% black, cast to lower-right per the top-left light) is baked into every elevated sprite — one convention, not per-artist.

5 · Collision

Each tile/object carries a collision flag (or mask for multi-tile): buildings, fences, water, trees and boulders are solid; paths, plaza, grass, decals are walkable. The town is laid out to support foot traffic, so the walkable network must stay connected.

6 · Autotile model

Surfaces & seams → 16-tile Wang (2-corner). Soft blended borders. Each set = 4 edges + 4 outer + 4 inner corners + diagonals. Every "transition" entry in the inventory expands to ~16 real tiles.
Roads & rivers → 16-piece connectivity. Crisp junctions: straight (H/V), corner, T, 4-way cross, end-cap. Drawn per rotation (light, above).
True tile count ≈ 300–500. The inventory lists each autotile SET as one entry; multiply sets out for the real backlog.

7 · Animation

Every animated entry needs frame count + fps + loop type specified at art time. Characters are 4-directional walk cycles × N frames (the most expensive assets — budget them first). Ambient motion (sway, flutter, water shimmer, mill wheel, fountain, smoke, glow) is 2–4 frames, looping.

8 · Season & day/night policy

Season: summer-only (for now). The board has its own seasonal engine; the town renders summer. If the town should reflect seasons later, the 12 ground terrains + trees + roofs need autumn/winter recolours — flagged, not yet specced.
Day/night: night layer in scope. A day/dusk state exists, so the FX · night layer (lit-window overlays, lamp light-pools, ambient tint) is required. If night is cut, drop that category.

9 · Naming & export

Atlas with padding/extrusion. Multi-tile sprites (e.g. 4×4 town hall = 128px) need 1–2px extrusion in the atlas to avoid bleeding at non-integer zoom.
Bitmask → filename. Autotile pieces named by their neighbour bitmask so the autotiler can map cell→tile deterministically.
One material = one folder. Group each material's base + ramp + transition set together; keep the inventory id as the asset id.
