Tiles Through the Seasons

Three farm tiles — wheat, a berry bush and a sunflower — each with a gentle idle loop and a full spring→summer→autumn→winter cycle, generated at both 64 px and 128 px.

A companion to Birch in the Breeze. Where the birch was drawn in code, these are made with the PixelLab generation MCP — one style-locked base sprite per subject, its seasons produced as edited states of that same sprite so the plant stays recognisable all year, then a short idle animation on each. The look is matched to the game's existing tile icons: soft rounded forms, smooth shading, a single soft dark outline and a warm, friendly palette. Concept only — nothing here is wired into the engine yet.

05How it was made

Where Birch in the Breeze was drawn in code, these three tiles were generated with the PixelLab pixel-art MCP — but with the same discipline: keep one recognisable subject and drive everything else from it.

  • One style-locked base per subject. Each tile starts from a single create_map_object in basic mode, with a fixed style recipe matched to the game's icons — “soft rounded shapes, smooth gentle shading, friendly clean cartoon style, soft dark outline, lit from upper-left”, a side view and a single-colour outline. The first sprite (summer wheat) was checked against the in-game tile icons before anything else was generated, so the whole set inherits one look.
  • Seasons are edited states, not new sprites. Spring, autumn and winter come from create_object_state on the summer base — “turn the leaves red and add ripe berries”, “dust it with snow”. Because each season is an edit of the same object, the plant keeps its silhouette and composition all year instead of being four unrelated drawings.
  • Both densities are native. 64 px and 128 px each have their own base + states — the 64s are drawn at 64, not shrunk from 128 — so the low-res versions get real pixel-art clustering rather than a blurry downscale.
  • Motion is a v3 idle loop. Each finished sprite gets one animate_object pass (mode v3, 8 frames) with a single gentle-breeze brief per subject: wheat sways and the heads nod, the bush settles and the berries jiggle, the sunflower head bobs on its stem. The base stays rooted so the wind moves through the sprite.
  • Generated sequentially. Sprites were produced subject-by-subject and verified against each other as they came back, so the three tiles read as one cohesive set rather than three separate experiments.

Concept only. The game still draws these tiles as static procedural icons. This is an exploration of what an animated, seasonal tile set could look like in the icon style — the GIFs live as external files under assets/seasonal-tiles/ and nothing here is wired into the engine.
